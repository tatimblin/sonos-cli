## Overview

Create `docs/references/sonos-api.md` ŌĆö a developer reference for the `sonos-api` crate that mirrors the format and depth of the existing `docs/references/sonos-sdk.md`. The CLI (non-TUI) path will call into `sonos-api` directly rather than routing through the high-level `sonos-sdk` facade, so this document is the build reference for Milestones 2ŌĆō5.

**Roadmap connection:** Supports Milestone 1 (foundation) and is the primary reference for Milestones 2ŌĆō5 (all CLI commands). The executor will call `SonosClient` operations directly.

---

## Problem Statement / Motivation

`sonos-sdk.md` covers the high-level wrapper crate ŌĆö `SonosSystem`, property handles, and the change-event model. The lower-level `sonos-api` crate is not documented anywhere in `sonos-cli/docs/`. This creates a gap:

- The CLI executor needs to call `SonosClient::execute_enhanced()` with the correct operation builders, request fields, and response types.
- Without a reference, developers must read the raw Rust source in `../sonos-sdk/sonos-api/src/` every time they add a command.
- Edge cases (bool serialisation as `"0"`/`"1"`, channel names, valid play mode strings, coordinator routing) are not captured anywhere in the `sonos-cli` repo.

---

## Proposed Solution

Write `docs/references/sonos-api.md` following the exact conventions of `sonos-sdk.md`:

- H1 title + one-paragraph scope statement
- Architecture diagram as a plain fenced block (no language tag)
- Numbered TOC with anchor links immediately after the architecture diagram
- Numbered H2 sections (`## 1.`, `## 2.`) with H3 subsections (`### 1.1`), separated by `---`
- Method signature tables with context-sensitive fourth column: `Return`/`Notes`/`Validation`/`Behaviour` as appropriate
- Primary structs rendered as fenced `pub struct` Rust blocks with inline comments; field tables used only for response types
- `┬¦N` shorthand for same-document cross-references
- Sentence case for all section titles; `Behaviour` (British spelling) in table column headers
- Fenced Rust code blocks for usage examples; `Return` (no "s") in action-table column headers, `Returns` in lookup-table headers
- A "See also" footer linking to `sonos-sdk.md` and the upstream spec

Also update `CLAUDE.md` to add a row in the Reference Documentation table for `sonos-api.md`.

### Revised section structure

Research identified that the original 15-section order followed crate internals rather than the executor author's workflow. The CLI executor author's questions in order are: (1) should I call this layer at all? (2) which service/operation? (3) what fields go in the request? (4) what do I read from the response? (5) what errors do I handle? The revised structure follows that flow.

Sections 3 (`UPnPOperation` trait internals), 5 (`Validate` trait), 11 (`ManagedSubscription` lifecycle), 12 (event framework), and 14 (utility functions) have been dropped or absorbed because the executor never implements those traits, never manages subscriptions directly, and never calls the utilities ŌĆö they are called internally by the operation structs.

### Critical behaviours block (content)

Placed before ┬¦1. One bold sentence + one consequence sentence per item, with `┬¦N` forward reference:

- **Bool fields serialise as `"0"`/`"1"`, not `"true"`/`"false"`.** Sending `"true"` causes the device to return `SoapFault(402)`. Affects `SetMute`, `SetLoudness`, `SetGroupMute`. See ┬¦6, ┬¦7.
- **`instance_id` is always `0`.** Injected into every request struct by the `define_upnp_operation!` macro; any other value returns `SoapFault(718)`. See ┬¦4.
- **`channel` must be `"Master"`, `"LF"`, or `"RF"`.** Validated by `validate_channel()` inside `build()`. Use `"Master"` in all normal cases; `"LF"`/`"RF"` only for stereo-paired speakers. See ┬¦6.
- **Play mode strings are raw UPnP, not Rust enum names.** `"NORMAL"`, `"REPEAT_ALL"`, `"REPEAT_ONE"`, `"SHUFFLE_NOREPEAT"`, `"SHUFFLE"`, `"SHUFFLE_REPEAT_ONE"`. See ┬¦5.
- **Group volume is `u16`, not `u8`.** `GroupRenderingControl` uses `desired_volume: u16` even though the valid range is 0ŌĆō100. See ┬¦7.
- **`GroupManagement` has no subscription helpers.** `group_management::operations` does not export `subscribe()` / `subscribe_with_timeout()`. Topology events arrive via `ZoneGroupTopology`. See ┬¦9.
- **Legacy execute path.** `SonosClient::execute()` (with `SonosOperation` trait) is retained for compatibility. All new code uses `execute_enhanced()` + `UPnPOperation`. See ┬¦2.

### Key missing items found during research (must include)

These were absent from the original plan but are present in the source and relevant to CLI developers:

- `SonosClient::with_soap_client(soap_client)` ŌĆö advanced constructor that bypasses the singleton `SoapClient`; document as an escape hatch
- `impl Default for SonosClient` ŌĆö delegates to `new()`; allows use in `#[derive(Default)]` structs
- `SonosClient::create_managed_subscription(ip, service, callback_url, timeout_seconds)` ŌĆö canonical lower-level subscription constructor; `subscribe()` and `subscribe_with_timeout()` delegate here; note default timeout is 1800 s
- `ServiceInfo::event_endpoint` ŌĆö the HTTP path for SUBSCRIBE/UNSUBSCRIBE; must appear in the `Service` endpoint table alongside `endpoint`
- `GroupManagement` scope is `PerCoordinator` ŌĆö add to the service table
- `OperationBuilder::build_unchecked()` ŌĆö bypasses validation; document with explicit warning against use in CLI executor
- `ValidationLevel` has exactly two variants: `None` and `Basic` (not `Comprehensive`)
- `ApiError::subscription_expired()` ŌĆö public constructor method, not a variant
- `From<ValidationError> for ApiError` ŌĆö `ValidationError` coerces to `ApiError::InvalidParameter` via `?`; document in ┬¦10 with function return type implication
- `DelegateGroupCoordinationToOperation::rejoin_group: bool` serialises as `"true"`/`"false"` (inconsistent with all other bool fields that use `"1"`/`"0"`)
- DIDL-Lite types (`DidlLite`, `DidlItem`, `DidlResource`) ŌĆö public structs used to parse `track_meta_data` strings from `GetPositionInfoResponse`; needed by `sonos status` executor branch
- `extract_xml_value(xml, tag) -> Option<String>` is a top-level crate re-export

### UPnP fault code table (for ┬¦10)

| Code | Name | When it occurs |
|------|------|----------------|
| 400 | Bad Request | Malformed SOAP envelope |
| 401 | Invalid Action | Action not implemented on this device model |
| 402 | Invalid Args | Wrong type, missing arg, or bool sent as `"true"`/`"false"` |
| 501 | Action Failed | Internal device error; transient state (e.g., play with empty queue) |
| 701 | Transition Not Available | State machine can't perform the action (e.g., pause when already paused) |
| 710 | Seek Mode Not Supported | Wrong `Unit` value in `SeekOperation` |
| 711 | Illegal Seek Target | Out-of-range seek position |
| 712 | Play Mode Not Supported | Invalid `NewPlayMode` string, or not coordinator |
| 718 | Invalid InstanceID | `instance_id` != 0 |
| 800 | Not coordinator | Operation sent to a non-coordinator speaker |
